## рҹҸӣпёҸ Patrones de DiseГұo

### 1. MVC (Model-View-Controller)
- **Models**: DefiniciГіn de entidades y lГіgica de dominio
- **Views**: Interfaces de usuario en Flet
- **Controllers**: Servicios que coordinan vistas y modelos

### 2. Repository Pattern
- AbstracciГіn de acceso a datos
- CRUD genГ©rico en BaseRepository
- Repositorios especГӯficos para cada entidad

### 3. Service Layer
- LГіgica de negocio centralizada
- Validaciones y reglas de negocio
- OrquestaciГіn de operaciones complejas

### 4. Dependency Injection
- InyecciГіn de sesiones de BD
- Facilita testing y mantenibilidad

## рҹ”җ Seguridad

### AutenticaciГіn
- Hash de contraseГұas con Bcrypt (factor 12)
- ValidaciГіn de credenciales en AuthService
- SesiГіn de usuario en memoria

### AutorizaciГіn
- Sistema de roles (Admin, Supervisor, Inspector, Consulta)
- Permisos basados en JSON
- Control de acceso en servicios

### ValidaciГіn de Datos
- Validaciones en capa de servicios
- Validadores especГӯficos (email, telГ©fono, RUC)
- SanitizaciГіn de entradas

## рҹ“Ғ Flujo de Datos

### Crear InspecciГіn
```
1. Usuario вҶ’ InspectionView.create()
2. InspectionView вҶ’ InspectionService.create_inspection()
3. InspectionService вҶ’ Validar datos
4. InspectionService вҶ’ InspectionRepository.create()
5. InspectionRepository вҶ’ DB.insert()
6. DB вҶ’ Return inspection_id
7. Return вҶ’ InspectionView (actualizar UI)
```

### Exportar a Excel
```
1. Usuario вҶ’ ReportView.export()
2. ReportView вҶ’ ExportService.export_inspections()
3. ExportService вҶ’ InspectionRepository.get_all(filters)
4. Repository вҶ’ DB.query()
5. ExportService вҶ’ Generate Excel (openpyxl)
6. ExportService вҶ’ Save file
7. Return вҶ’ ReportView (mostrar confirmaciГіn)
```

## рҹ—„пёҸ GestiГіn de Datos

### Base de Datos
- **Motor**: PostgreSQL 14+
- **ORM**: SQLAlchemy 2.0
- **Pool**: QueuePool (size=10, max_overflow=20)
- **Migraciones**: Alembic (opcional)

### Transacciones
- Commit automГЎtico en operaciones exitosas
- Rollback en excepciones
- Context managers para transacciones complejas

### ГҚndices
- Claves primarias (id)
- Claves forГЎneas
- ГҚndices en campos de bГәsqueda frecuente
- ГҚndices Гәnicos en cГіdigos

## рҹ“Ұ MГіdulos Principales

### app/config

- Propiedades calculadas

### app/repositories

### app/services

- Vistas Flet para cada mГіdulo
- Componentes de UI reutilizables

### app/utils
- `validators.py`: Validadores
- `formatters.py`: Formateadores
- `logger.py`: Sistema de logging

### Horizontal
